
## 2. 「器の整理 vs 器の拡張」分類プロンプト（企画ストック棚卸し）

```
以下の企画リストを「器の整理（既存価値の最適化）」と「器を大きくする（非連続な拡張）」に分類してください。

判定基準:
- 器の整理: タイトル改善、内部リンク強化、リライト、SEOキーワード調整など。CVRやPVを数%動かす。
- 器の拡張: 新カテゴリ開拓、新フォーマット、新キーワード群、新セグメントへの進出、新シリーズ立ち上げ。

# 企画リスト
{ここに企画タイトルと一行の概要を箇条書きで}

# 出力
- 各企画にフラグ（整理 / 拡張）
- 拡張比率（%）
- 拡張比率が30%未満なら警告と追加企画3案
```

## 9. リライト判定プロンプト（過去記事棚卸し）

```
以下の過去記事について、「削除 / 統合（既存記事Xとマージ）/ リライト」の3択判断とその根拠を出してください。

# 入力
- 記事タイトル: {}
- 公開日: {}
- 直近3ヶ月のPV: {}
- 直近3ヶ月のCVR: {}
- 検索流入キーワード: {}
- 関連既存記事: {}

# 出力
- 判断（削除 / 統合 / リライト）
- 根拠（3行）
- 統合の場合: マージ先記事と統合方針
- リライトの場合: 改訂ポイント3つ
- 削除の場合: noindex化 or 完全削除の判断
```
