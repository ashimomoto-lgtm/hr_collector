# 運用チェックリスト

実務で繰り返し使うチェックリスト集。Notion / スプレッドシートに転記して使う。

---

## 企画前チェックリスト（新記事）

- [ ] **UVP（20字以下）** が言語化できているか
- [ ] 「器の中の整理」か「器を大きくする」かタグ付けされているか
- [ ] **Profit Tree** のどのノードを動かす企画か明示されているか
- [ ] **Ultimate Goal**（1on1総研の3年Goal）と紐づいているか
- [ ] 競合HRメディアにはない価値が3文で書けるか
- [ ] 想定読者にインタビュー（最低1名）して課題が確認されているか
- [ ] 「自社宣伝」「カテゴリ啓蒙」のどちらかタグ付け（後者50%以上）
- [ ] 事前数値仮説（予想PV / 滞在時間 / CVR）を記録したか
- [ ] 短期施策か中期MOAT（ノウハウ蓄積）か明示されているか
- [ ] 「読者の中で立ち上がる問い」が1文で書けるか

## 公開前チェックリスト（記事）

- [ ] **Structure**: User Flow Diagram を1枚描いたか（読者の情報探索フロー）
- [ ] **Skeleton**: 目次が情報フローに沿っているか
- [ ] **最高/最悪フロー**: 想定読者の最高フローと最悪フローを描いたか
- [ ] **オンボーディング**: 初訪問者向けのナビゲーション設計があるか（最初の3記事への動線）
- [ ] **問い**: リード文に読者の問いが立ち上がる仕掛けがあるか
- [ ] **CTA**: アクティベーション設計がされているか（PSF/PMF視点）
- [ ] **内部リンク**: Structure レイヤーで意図的に配置されているか
- [ ] **公開前ユーザーテスト**: 重要企画は3〜5名で実施したか
- [ ] **タイトル**: 検索意図への答えだけでなく、未言語化の問いを掘れているか

## 月次振り返りチェックリスト（編集会議）

### KPI 棚卸し
- [ ] 週間 PV TOP 5 / BOTTOM 5 をリスト化
- [ ] 離脱率 高い記事 TOP 3 を特定
- [ ] 資料 DL 数 TOP 3 の流入元・滞在時間・シェア率を比較（因果関係の3パターンで分類）
- [ ] 仮説 vs 実績の照合（前月企画の予想PVと実績）

### 引き算判断
- [ ] 過去2年公開記事の下位 20% を「削除 / 統合 / リライト」で3択判断
- [ ] 不要なカテゴリ・タグ・ナビ要素を3つ以上削除候補に
- [ ] 6ヶ月以上前の流入減少記事を改訂優先度順にリスト化

### 戦略アップデート
- [ ] **Profit Tree** を更新し、次月どのノードを動かすか合意
- [ ] **器の整理 vs 器の拡張** の企画比率（後者30%以上か）
- [ ] **カテゴリ啓蒙 vs 自社宣伝** の比率（前者50%以上か）
- [ ] **短期 vs 中期MOAT** の施策バランス
- [ ] AIネイティブ競合の動向監視（自己破壊シナリオの更新）

### チーム
- [ ] **記事Retro**（公開後48h）: 全記事で実施されたか
- [ ] **週次Retro**: フローのボトルネック共有
- [ ] **月次チーム関係性Retro**: 実施
- [ ] **育成6領域**で最弱領域を特定し勉強会枠を確保
- [ ] **事例ストック**: 各メンバーが週1以上シェアしたか
